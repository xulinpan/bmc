ACTIONS REQUIRED BEFORE WEB SUBMISSION
1. bmcart.cls: main.tex uses the official BMC class, which is NOT in standard TeX
   distributions. Obtain bmcart.cls (and bmc-mathphys.bst if you switch to BibTeX) from the
   BMC / Overleaf "BioMed Central article" template and place it next to main.tex, or just
   upload main.tex to the BMC submission system (it provides the class). Use main_article.tex
   only for local previewing.
2. Open code/data (MANDATORY before acceptance): the Availability statement currently states the
   code is provided as supplementary material and "intended for deposition in a public repository
   before final journal submission." BMC requires the actual deposit: create a public GitHub repo,
   mint an archival DOI (Zenodo) from a tagged release, add an OSI license, and update the
   Availability statement to cite the URL + DOI. "Available upon request" is not acceptable.
3. Confirm author affiliation and corresponding-author email.
4. Confirm the NSFC grant number 11461079 is formatted exactly as the funder requires; note
   the affiliation (Penn State) and funder (NSFC, China) pairing in case the editor queries it.
5. Upload each figure as a separate graphics file if the submission system requests it; for the
   final BMC build you may uncomment the two \def\includegraphics{} lines in main.tex so figures
   are not embedded.
6. Upload supplementary scripts/metrics as Additional Files only if requested by the portal.
